**服务器环境构建尝试：**

## 🚀 方案1：使用Android Studio（推荐）

### 步骤1：下载Android Studio

1. 访问 https://developer.android.com/studio
2. 下载适合你系统的版本（Windows/Mac/Linux）
3. 安装并启动

**安装时间：** 20-40分钟
**下载大小：** 约1GB

### 步骤2：打开项目

1. 启动Android Studio
2. 选择 "Open an Existing Project"
3. 导航到 `HabitTracker` 目录
4. 点击 "OK"

### 步骤3：同步Gradle

1. 等待Android Studio自动同步Gradle
2. 如果提示缺少SDK组件，点击安装
3. 等待下载完成（可能10-30分钟）

### 步骤4：构建Debug APK

**快速构建（用于测试）：**

1. 选择 `Build` → `Build Bundle(s) / APK(s)` → `Build APK(s)`
2. 等待构建完成
3. 构建完成后，点击通知中的 "locate"
4. APK位置：`app/build/outputs/apk/debug/app-debug.apk`

### 步骤5：构建Release APK（用于上架）

**创建签名Keystore：**

1. 菜单：`Build` → `Generate Signed Bundle / APK`
2. 选择 "APK"
3. 点击 "Create new..."

4. 填写Keystore信息：
   - **Keystore path**: 选择保存位置
   - **Password**: 设置密码（记住！）
   - **Key alias**: 输入别名（如：release）
   - **Key password**: 设置密码
   - **Validity (years)**: 25
   - **Certificate**: 填写你的信息

5. 点击 "OK"

**构建Release APK：**

1. 选择刚创建的Keystore
2. 选择 "release" 构建变体
3. 点击 "Finish"
4. 等待构建完成

5. APK位置：`app/build/outputs/apk/release/app-release.apk`

## 🔧 方案2：使用命令行Gradle（进阶）

### 环境要求

- Java 17+
- Android SDK
- Gradle 8.0+

### 构建步骤

**1. 设置环境变量：**

```bash
# Linux/Mac
export JAVA_HOME=/path/to/java17
export ANDROID_HOME=/path/to/android-sdk
export PATH=$JAVA_HOME/bin:$ANDROID_HOME/cmdline-tools/latest/bin:$ANDROID_HOME/platform-tools:$PATH

# Windows
set JAVA_HOME=C:\path\to\java17
set ANDROID_HOME=C:\path\to\android-sdk
set PATH=%JAVA_HOME%\bin;%ANDROID_HOME%\cmdline-tools\latest\bin;%ANDROID_HOME%\platform-tools;%PATH%
```

**2. 构建Debug APK：**

```bash
cd HabitTracker
./gradlew assembleDebug
```

**3. 构建Release APK：**

首先，在 `app/build.gradle` 添加签名配置：

```gradle
android {
    signingConfigs {
        release {
            storeFile file("/path/to/your/keystore.jks")
            storePassword "your_keystore_password"
            keyAlias "your_key_alias"
            keyPassword "your_key_password"
        }
    }

    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled false
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
}
```

然后构建：

```bash
./gradlew assembleRelease
```

**APK位置：**
- Debug: `app/build/outputs/apk/debug/app-debug.apk`
- Release: `app/build/outputs/apk/release/app-release.apk`

## 📦 方案3：使用GitHub Actions（自动化）

### 创建工作流文件

在项目根目录创建 `.github/workflows/build-apk.yml`:

```yaml
name: Build Android APK

on:
  push:
    branches: [ main ]
  workflow_dispatch:

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v3

    - name: Set up JDK 17
      uses: actions/setup-java@v3
      with:
        java-version: '17'
        distribution: 'temurin'

    - name: Setup Android SDK
      uses: android-actions/setup-android@v2

    - name: Grant execute permission for gradlew
      run: chmod +x gradlew

    - name: Build Debug APK
      run: ./gradlew assembleDebug

    - name: Upload Debug APK
      uses: actions/upload-artifact@v3
      with:
        name: debug-apk
        path: app/build/outputs/apk/debug/app-debug.apk
```

### 使用方法

1. 推送代码到GitHub
2. 在GitHub Actions页面运行工作流
3. 下载生成的APK

## 🐛 常见问题排查

### 问题1：Gradle同步失败

**错误信息：**
```
Could not resolve all dependencies for configuration ':app:debugRuntimeClasspath'.
```

**解决方案：**
1. 检查网络连接
2. 使用VPN（如果在中国）
3. 配置国内镜像源

在 `build.gradle` 添加：

```gradle
repositories {
    maven { url 'https://maven.aliyun.com/repository/public' }
    maven { url 'https://maven.aliyun.com/repository/google' }
    google()
    mavenCentral()
}
```

### 问题2：SDK未找到

**错误信息：**
```
SDK location not found. Define location with an ANDROID_HOME environment variable.
```

**解决方案：**
在 `local.properties` 添加：
```properties
sdk.dir=/path/to/android-sdk
```

### 问题3：依赖下载慢

**解决方案：**
1. 配置Gradle使用国内镜像
2. 增加Gradle内存限制
3. 使用离线模式（如果依赖已下载）

在 `gradle.properties` 添加：
```properties
org.gradle.jvmargs=-Xmx2048m
org.gradle.daemon=true
```

### 问题4：构建成功但APK无法安装

**可能原因：**
- 签名问题
- SDK版本不兼容
- 权限配置错误

**解决方案：**
- 重新签名APK
- 检查 `minSdkVersion` 设置
- 验证 `AndroidManifest.xml` 中的权限

## 💡 最佳实践

### 1. 使用版本控制

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/你的用户名/HabitTracker.git
git push -u origin main
```

### 2. 创建多个构建变体

```gradle
android {
    buildTypes {
        debug {
            applicationIdSuffix ".debug"
            debuggable true
        }
        release {
            minifyEnabled true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
        beta {
            applicationIdSuffix ".beta"
            debuggable false
            minifyEnabled false
        }
    }
}
```

### 3. 自动版本管理

```gradle
android {
    defaultConfig {
        versionCode System.getenv("BUILD_NUMBER")?.toInteger() ?: 1
        versionName "1.0.${versionCode}"
    }
}
```
